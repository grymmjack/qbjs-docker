# Tauri wrapper for a QBJS app

This is the **modern** "ready-to-run exe" path (the NWJS replacement). Tauri wraps
your compiled QBJS web bundle (`../dist`) in a native window using the operating
system's own webview, so binaries are a few MB instead of ~100 MB.

## Layout

```
tauri/
├── package.json          # provides @tauri-apps/cli
└── src-tauri/
    ├── Cargo.toml        # Rust crate (tiny release profile)
    ├── build.rs
    ├── tauri.conf.json   # window + bundle config; frontendDist = ../dist
    ├── src/main.rs       # loads the webview, no app logic
    └── icons/            # generated by `npm run tauri icon <png>`
```

## Build locally

```bash
# 1) Produce the web bundle first (from your project root):
qbjs-build.sh main.bas --name "My App" --out tauri/dist

# 2) Generate icons from any square PNG (once):
cd tauri
npm install
npm run tauri icon ../dist/logo-256.png

# 3) Build the installer for the current OS:
npm run tauri build
# Output: src-tauri/target/release/bundle/**  (.exe/.msi, .AppImage/.deb, .dmg)
```

## Requirements

- Rust toolchain (`rustup`)
- **Linux only:** `libwebkit2gtk-4.1-dev`, `libappindicator3-dev`, `librsvg2-dev`, `patchelf`
- Windows/macOS: no extra system deps beyond Rust + the platform SDK

In CI this is all handled for you by `tauri-apps/tauri-action` — see the
`reusable-build.yml` workflow in the repo root.

## Window defaults

Edit `src-tauri/tauri.conf.json` → `app.windows[0]` to change the title, size,
`resizable`, or `fullscreen`. For a fixed-resolution retro game, set `resizable`
to `false` and pick a `width`/`height` that matches your `SCREEN` mode.
